# Clips API and Web UI

The EMD Agent now includes a comprehensive API for browsing and playing recorded motion detection clips, along with an embedded web UI.

## 📚 API Documentation

**Interactive API Documentation:** [http://localhost:8080/docs](http://localhost:8080/docs)

The agent includes embedded **Swagger UI** for interactive API exploration:
- Browse all endpoints
- View request/response schemas
- Try API calls directly from the browser
- Download OpenAPI 3.0 specification

**OpenAPI Spec:** [http://localhost:8080/docs/openapi.json](http://localhost:8080/docs/openapi.json)

## API Endpoints

### List Clips

**GET** `/api/clips`

List all recorded clips across all cameras.

**Query Parameters:**
- `camera` (optional) - Filter clips by camera name

**Example:**
```bash
# List all clips
curl http://localhost:8080/api/clips | jq

# List clips for specific camera
curl http://localhost:8080/api/clips?camera=axis_82_2 | jq
```

**Response:**
```json
{
  "clips": [
    {
      "camera": "axis_82_2",
      "filename": "20260518_091523_motion.ts",
      "path": "/var/lib/emd-agent/clips/axis_82_2/20260518_091523_motion.ts",
      "size": 2456789,
      "mod_time": "2026-05-18T09:15:23Z",
      "url": "/api/clips/axis_82_2/20260518_091523_motion.ts"
    }
  ],
  "total": 1
}
```

### Serve Clip File

**GET** `/api/clips/{camera}/{filename}`

Serve a specific video clip file for playback.

**Features:**
- Proper MIME types for video streaming (`video/mp2t`, `video/mp4`, `video/x-matroska`)
- HTTP Range request support for seeking
- Browser-compatible streaming

**Example:**
```bash
# Stream a clip
curl http://localhost:8080/api/clips/axis_82_2/20260518_091523_motion.ts --output clip.ts

# Or open directly in browser
open http://localhost:8080/api/clips/axis_82_2/20260518_091523_motion.ts
```

**Security:**
- Path traversal protection (`.` not allowed in camera names or filenames)
- Files must be within the configured `clip_root` directory
- Only video files are served (`.ts`, `.mp4`, `.mkv` extensions)

### Web UI

**GET** `/`

Access the embedded web UI for browsing and playing clips.

**Features:**
- 📋 List all clips with metadata (size, timestamp, camera)
- 🎥 Built-in HTML5 video player
- 🔍 Filter clips by camera
- 📊 Statistics (total clips, total size, camera count)
- 🔄 Auto-refresh every 30 seconds
- 📱 Responsive design

**Access:**
```bash
# Open in browser
open http://localhost:8080/
```

## Browser Compatibility

### MPEG-TS Files

The `.ts` (MPEG-TS) files generated by the agent are playable in modern browsers:

| Browser | MPEG-TS Support | Notes |
|---------|----------------|-------|
| **Chrome/Edge** | ✅ Native | Direct playback via HTML5 video |
| **Firefox** | ✅ Native | Direct playback via HTML5 video |
| **Safari** | ✅ Native | Best support (HLS origin) |

**Important:** MPEG-TS files must have:
- Valid SPS/PPS parameter sets (injected by agent)
- Proper codec initialization
- Correct MIME type: `video/mp2t`

The agent automatically handles these requirements.

### MP4 Files

If you configure `container = "mp4"` in the config:
- Universal browser support
- Better seeking performance
- Larger file sizes (due to additional container overhead)

## Configuration

Enable the API server in `agent.toml`:

```toml
[runtime]
clip_root = "/var/lib/emd-agent/clips"
```

Start the agent with API enabled:

```bash
./emd-agent --config config.toml --api :8080
```

## Usage Examples

### List Recent Clips

```bash
curl http://localhost:8080/api/clips | jq '.clips[] | {camera, filename, size}'
```

### Download a Clip

```bash
curl http://localhost:8080/api/clips/axis_82_2/20260518_091523_motion.ts \
  -o motion_event.ts
```

### Embed in Custom UI

```html
<video controls width="640">
  <source src="http://localhost:8080/api/clips/axis_82_2/20260518_091523_motion.ts"
          type="video/mp2t">
  Your browser does not support the video tag.
</video>
```

### Fetch Clip List from JavaScript

```javascript
async function loadClips() {
  const response = await fetch('/api/clips');
  const data = await response.json();
  
  data.clips.forEach(clip => {
    console.log(`${clip.camera}: ${clip.filename} (${clip.size} bytes)`);
  });
}
```

## Performance Considerations

### File Serving

- Files are streamed directly from disk (no buffering in memory)
- HTTP Range requests enable efficient seeking
- Cache headers reduce repeated transfers
- Only video file extensions are served

### Clip Scanning

- Clips are scanned on-demand (not cached)
- Directory traversal is limited to clip_root
- Results are sorted by modification time (newest first)
- Filter by camera reduces scan scope

### Scalability

For large deployments (thousands of clips):

1. **Enable filtering** - Use `?camera=` query parameter to reduce scan scope
2. **Implement pagination** - Future enhancement for limiting results
3. **Use external CDN** - Serve clips from object storage for better performance
4. **Monitor disk I/O** - Scanning large directories can impact disk performance

## Security

### Path Traversal Protection

The API validates all file paths to prevent directory traversal attacks:

```bash
# These are blocked:
curl http://localhost:8080/api/clips/../../../etc/passwd  # 400 Bad Request
curl http://localhost:8080/api/clips/camera1/../../../etc/passwd  # 400 Bad Request
```

### Authentication

**Current:** No authentication (local/trusted network only)

**Production:** Add authentication middleware:
- API keys in `Authorization` header
- JWT tokens
- mTLS for pod-to-pod communication
- Network policies to restrict access

### CORS

If accessing from a different origin, add CORS headers:

```go
w.Header().Set("Access-Control-Allow-Origin", "*")
w.Header().Set("Access-Control-Allow-Methods", "GET, OPTIONS")
```

## Troubleshooting

### Clips Not Showing

1. **Check clip_root path:**
   ```bash
   ls -la /var/lib/emd-agent/clips/
   ```

2. **Verify agent has recorded clips:**
   ```bash
   curl http://localhost:8080/api/clips | jq '.total'
   ```

3. **Check permissions:**
   ```bash
   # Agent needs read access to clip directory
   sudo chown -R emd-agent:emd-agent /var/lib/emd-agent/clips/
   ```

### Video Won't Play

1. **Check file extension:**
   - Browser must recognize: `.ts`, `.mp4`, `.mkv`
   - Check MIME type in response headers

2. **Verify codec compatibility:**
   - MPEG-TS: H.264 or H.265 video only
   - Must have SPS/PPS parameter sets (agent injects these automatically)

3. **Test with curl:**
   ```bash
   curl -I http://localhost:8080/api/clips/camera1/clip.ts
   # Should show: Content-Type: video/mp2t
   ```

4. **Check browser console for errors:**
   - Open Developer Tools → Console
   - Look for network or decode errors

### 404 Not Found

1. **Verify filename is exact match:**
   ```bash
   # List clips to get exact filename
   curl http://localhost:8080/api/clips?camera=axis_82_2 | jq '.clips[].filename'
   ```

2. **Check URL encoding:**
   ```bash
   # Special characters must be URL-encoded
   curl http://localhost:8080/api/clips/camera1/clip%20with%20spaces.ts
   ```
